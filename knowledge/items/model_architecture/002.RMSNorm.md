## $RMSNorm$

$RMSNorm$ 就是对每个向量单独按它自己的整体大小做归一化，再乘上一个可学习的缩放参数 $weight$。它不看同一个 $batch$ 里的其他向量，也不看别的 $token$。

在 $Transformer$ 里，如果张量形状是 $batch, seq_len, dim$，那么通常：$x[b,t,:]$这一整个长度为 $dim$ 的向量，就表示第 $b$ 个样本里、第 $t$ 个 $token$ 的隐藏表示。所以 $RMSNorm$ 就是“对每个 $token$ 的隐藏向量单独做归一化。”

一个向量对应一个 $token$ 是 $NLP \space Transformer$ 里最常见的情况；如果是图像、语音或别的模态，也可能对应的是一个 $patch$、$frame$ 或别的序列位置。


设一个向量是：

$$x = [x_1, x_2, \dots, x_d]$$

$RMS$ 就是 $Root Mean Square$，即均方根。 $RMSNorm$ 的计算就是先计算 $RMS$：

$$RMS(x) = \sqrt{\frac{1}{d}\sum_{i=1}^d x_i^2 + \epsilon}$$

然后做归一化：

$$RMSNorm(x) = weight \odot \frac{x}{RMS(x)}$$

其中：
- $d$ 是最后一维的长度
- $\epsilon$ 是防止除零的小常数
- $weight$ 是可学习参数，形状通常是 $$[dim]$$


## $RMSNorm$ 和 $LayerNorm$ 的区别

$LayerNorm$ 会做两件事：

1. 减去均值
2. 除以标准差

公式是：

$$LayerNorm(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

而 $RMSNorm$ 更简单，它不减均值，而且只按均方根缩放

所以可以理解为 $LayerNorm$ 既管“偏移”，也管“尺度”；而 $RMSNorm$ 只管“尺度”

## $RMSNorm$ 实现代码

```python
import torch
import torch.nn as nn

class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def _norm(self, x):
        rms_inv = torch.rsqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return x * rms_inv

    def forward(self, x):
        x_norm = self._norm(x.float()).type_as(x)
        return x_norm * self.weight

x = torch.randn(2, 4, 8)  # [batch, seq, hidden]
norm = RMSNorm(8)
y = norm(x)

print(x.shape)  # torch.Size([2, 4, 8])
print(y.shape)  # torch.Size([2, 4, 8])
```

## 一个具体例子

假设输入是：

$$x \in R^{[2,3,4]}$$

即 $batch = 2$, $seq\_len = 3$, $dim = 4$

这种情况下一共有 $$2 \times 3 = 6$$ 个向量，每个向量长度是 $$4$$。

现在只看其中一个向量：

$$x[0,0,:] = [1,2,3,4]$$

该向量的计算过程如下：
1. 平方

$$[1,2,3,4]^2 = [1,4,9,16]$$

2. 求平均

$$mean([1,4,9,16]) = \frac{1+4+9+16}{4} = 7.5$$

3. 开根号，得到 $RMS$

$$RMS(x[0,0,:]) = \sqrt{7.5} \approx 2.7386$$

4. 原向量除以这个 $RMS$

$$\frac{[1,2,3,4]}{2.7386} \approx [0.365, 0.730, 1.095, 1.461]$$

如果此时：

$$weight = [1,1,1,1]$$

那么输出就是：

$$[0.365, 0.730, 1.095, 1.461]$$



## 和 $attention$ 的形状关系

在多头注意力里，通常流程是：

$$
[batch, seq\_len, dim]
\rightarrow
[batch, seq\_len, num\_heads, head\_dim]
\rightarrow
[batch, num\_heads, seq\_len, head\_dim]
$$

每个 $head$ 分别做 $attention$ 之后，结果通常会再拼回去：

$$
[batch, num\_heads, seq\_len, head\_dim]
\rightarrow
[batch, seq\_len, num\_heads, head\_dim]
\rightarrow
[batch, seq\_len, dim]
$$

其中：

$$dim = num\_heads \times head\_dim$$

之后再进入后续模块，比如 $RMSNorm$、残差连接、$MLP$ 等。


## 总结

$RMSNorm$ 就是对每个向量求该向量自己的 $RMS$， 再用这个 $RMS$ 去缩放自己

## Q&A
1. 对于输入 $x.shape = [batch, seq\_len, dim]$, $RMSNorm$ 到底是对哪一维做归一化？如果把输入换成多头注意力中的 $x.shape = [batch, num\_heads, seq\_len, head_dim]$, 你觉得应该对哪一维做归一化，为什么？

    代码实现中是`x.pow(2).mean(-1, keepdim=True)`，所以说是对最后一维做归一化。简单来说就是 $RMSNorm$ 通常总是对“特征维”做归一化，也就是最后一维，不会跨 $batch$，不会跨 $seq\_len$，通常也不会跨 $num\_heads$

2. 为什么 $RMSNorm$ 只看 $\sqrt{mean(x^2)}$ 而不减均值？“只控制尺度、不控制中心”这件事，在表达能力和训练稳定性上分别意味着什么？

    $LayerNorm$ 是 $\frac{x - \mu}{\sqrt{\sigma ^ 2 + \epsilon}}$，$RMSNorm$ 是 $\frac{x}{\sqrt{mean(x^2) + \epsilon}}$。区别在于 $LayNorm$ 去掉了均值偏移和整体尺度， $RMSNorm$ 只去掉整体尺度。在表达能力上，“只控制尺度、不控制中心”意味着向量整体偏正、偏负这件事会保留下来，模型可以继续利用这个“偏移信息”，有时候这反而是好事，因为残差流里可能确实携带了有用的偏移信息，$RMSNorm$ 不会强行把它抹掉。在训练稳定性上，很多时候，真正容易失控的是“幅值越来越大或越来越小”，而不是“均值不是 0”，$RMSNorm$ 只管幅值，已经足够解决很多深层网络里的数值放大问题。

3. 证明或解释下面两种写法为什么等价：1) `x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + eps)` 2)`x / torch.sqrt(x.pow(2).mean(-1, keepdim=True) + eps)`. 实现里更常见前者，你觉得为什么？

    等价的，因为$rsqrt(a)=\frac{1}{\sqrt a}$。前者更常见是因为 $rsqrt$ 在底层常有更直接的实现，效率更好; 少一次显式除法，通常更适合硬件优化。

4. 代码里为什么要写：`self._norm(x.float()).type_as(x)`而不是直接`self._norm(x)`。这里解决的是“精度问题”还是“梯度问题”还是“两者都有”？请尽量具体一点。
   
    因为 $RMSNorm$ 里的操作有平方、求平均、开根号、取倒数，这些在低精度下都容易出问题，尤其是 $float16$，解决的方法就是先把它转成 $float32$ (对应`x.float()`)，用 $float32$ 算归一化，再转换回原本的$dtype$ （对应`.type_as(x)`）。这里解决的主要是 $forward$ 计算过程中的精度问题，但也会间接改善梯度，因为梯度来自前向图，前向图更稳定，反向通常也更稳定。

5. 如果输入是 $float16$，而且某些值很大，下面哪一步最容易出数值问题？1）`x.pow(2)`; 2)`x.pow(2).mean(...)`; 3)`torch.sqrt(...)`. 为什么很多实现会先转 $float32$，但最后再转回原 $dtype$？
   
   参考4，如果某些值很大，第一步就很容易变成 $inf$，把 $inf$ 传下去得到的归一化值可能变成0或 $nan$。经过归一化计算之后结果已经比较稳定了，这时候转会原 $dtype$, 后续的算子可以用低精度跑，更省显存

6. 为什么 $eps$ 要加在这里：`mean(x^2) + eps`.而不是加在别的位置？比如下面这些写法分别有什么问题或差异？1)`torch.rsqrt(x.pow(2).mean(-1, keepdim=True)) + eps`; 2)`torch.rsqrt(x.pow(2).mean(-1, keepdim=True) ) * (1 + eps)`

    因为加 $eps$ 是为了稳定分母，如果 $mean(x^2)$ 本身很小甚至为0，加上 $eps$ 可以让分母有个下界，不至于趋近于0。后面两种做法都没有做到稳定分母，所以都不对。简单来说就是 $eps$ 必须加在“开根号之前、分母内部”，才真的是数值稳定项。

7. 如果某个 $token$ 的向量刚好全 0，即 $x=[0,0,…,0]$。$RMSNorm$ 输出是什么？梯度行为会有什么需要注意的地方？
   
   $mean(x^2)=0$，所以 $RMSNorm(x)=\frac{x}{mean(x^2) + \epsilon} = \frac{x}{\epsilon} = 0$, 所以输出仍然为0。此时$y = \frac{x}{\epsilon}$，所以$y' = \frac{1}{\epsilon}$，如果 $\epsilon = 1e-5$，那么这个值大概是 316 左右，不是无限大也不会很小。（所以 $\epsilon$ 不是越小越好，太小的话会让这里的局部梯度变大）

8. $RMSNorm$ 里的参数通常只有 $weight$, 为什么很多实现没有 $bias$？如果你强行加一个 $bias$，它和 $LayerNorm$ 的差别会缩小吗，还是本质仍然不同？
   
   $RMSNorm$ 和 $LayerNorm$ 的一个重要区别就是它接受均值不一定是0。加 $bias$ 会让它看起来更像 $LayerNorm$，但不会让它变成 $LayerNorm$

9.  假设把`mean(-1)`改成`sum(-1)`同时别的地方不变，会发生什么？这相当于只是差一个常数吗？这个常数和 $dim$ 有什么关系？

    $RMS(x)=\sqrt{\frac{1}{d} \sum{x_i^2}}=\frac{1}{\sqrt{d}}\sqrt{\sum{x_i^2}}$，如果改成`sum(-1)`则变成 $\sqrt{\sum x_i^2}$,所以使用`sum(-1)`时，输出会比使用`mean(-1)`少一个 $\sqrt{d}$ 的因子，这会导致输出的尺度会依赖于$dim$, $dim$ 变了归一化之后的幅值也会变，用`mean(-1)`则把不同维度长度的行为统一了，所以工程上更喜欢用`mean(-1)`

10. 下面这句里，为什么 $weight$ 放在最后乘？`return self.weight * self._norm(...)`.如果把 $weight$ 提前到归一化之前：`self._norm(self.weight * x)`.二者等价吗？为什么这题很关键？
    
    不等价。标准写法是$y = weight \odot \mathrm{Norm}(x)$，也就是先对 $x$ 做 $RMSNorm$，再用 $weight$ 对每个维度做逐元素缩放。此时，$weight$ 只是对“已经归一化好的结果”做一次可学习的重新标定。换句话说，归一化负责把整体尺度控制住，$weight$ 负责告诉模型“哪些维度该放大，哪些维度该缩小”。如果改成：$y = \mathrm{Norm}(weight \odot x)$情况就变了，因为这时 $weight$ 不仅影响分子，还会进入分母。对 $RMSNorm$ 来说，分母是：$RMS(weight \odot x) = \sqrt{\frac{1}{d}\sum_i (weight_i x_i)^2 + \epsilon}$，这意味着每个维度的 $weight$ 都会参与整体 $RMS$ 的计算，从而影响所有维度的归一化结果，$weight$ 的作用不再是“独立地调每一维”，而是变成了“先改变整个向量的形状，再按这个新形状重新归一化”，这样一来各个维度之间就耦合起来了。所以，标准写法里的 $weight$ 语义是先统一归一化，再逐维缩放；而提前乘的写法更像是先扭曲输入，再重新定义整体尺度。
    
    只有在很特殊的情况下两者才近似等价，比如 $weight$ 不是逐维参数，而只是一个标量，或者所有维度上的 $weight$ 都相同。因为这时只是对整个向量做统一缩放，而这种统一缩放大多会被归一化抵消掉。

11. 你觉得 $RMSNorm $是否满足下面这个性质：如果把输入整体放大为 $cx$ 输出是否基本不变？也就是：$\operatorname{RMSNorm}(cx) \stackrel{?}{=} \operatorname{RMSNorm}(x)$.什么时候近似成立，什么时候不严格成立？

    $RMSNorm$ 对输入的正比例整体缩放近似不变，因为分子和分母会同时按相同尺度变化，彼此抵消；但这种性质只是在 $\epsilon$ 可以忽略时近似成立。若输入幅值较小，$\epsilon$ 不能忽略，则不再严格成立。若缩放系数 $c$ 为负，还会额外带来一个整体符号翻转。

14. 下面这个实现有没有隐藏的小坑？请至少说出 2 个你会想检查的点。
    
```python
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-5):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(dim))
        self.eps = eps

    def forward(self, x):
        rms = x.pow(2).mean(-1, keepdim=True)
        x = x * torch.rsqrt(rms + self.eps)

```
